Tutorial - Portas em um projeto MultiSEL


As portas são responsáveis por disponibilizar uma interface de acesso para uma mídia ou ES dentro de uma cena. No AMUSE VR usam-se as portas para a apresentação mudar de uma cena para outra através de uma relacionamento.

Portas

A imagem abaixo mostra na linha 41 um exemplo porta.



Neste exemplo, a cena 2 possui duas mídias do tipo imagem e uma porta que aponta para uma das imagens componentes, conforme a ilustração abaixo.



Para criar uma porta é necessário criar a tag port, com um atributo identificador único “id” e o atributo component, que representa a mídia acessada pela porta, como na linha 41.

No AMUSE VR, todo relacionamento (tag relation) externo às cenas necessita de uma porta para dar acesso ao conteúdo da cena, como ilustrado na figura abaixo.



O relacionamento ilustrado na figura acima é definido conforme o código MultiSEL a seguir nas linhas 75 a 78, ou seja, quando a mídia imagem indicada pela porta da cena 2 for clicada, a cena 1 será exibida.



Atenção

Para criar relacionamentos entre cenas diferentes, a tag relation deve ser definida dentro do corpo do documento tag body.
